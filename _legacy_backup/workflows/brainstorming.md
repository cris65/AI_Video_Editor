# Brainstorming Mode

**Command:** `/brainstorming`

**Description:** Attiva una sessione pura di ideazione, strategia e validazione concettuale con il Tech Lead.

**Regole di Ingaggio (STRICT):**
1. **BLOCCO TOTALE DEI FILE (ZERO WRITE):** Sotto nessuna circostanza ti è permesso modificare, creare o cancellare file nel repository (niente `write_to_file`, niente `multi_replace_file_content`, niente `sed`).
2. **MODALITÀ DIALOGO:** Devi agire strettamente come partner strategico, restituendo le idee organizzate e facendo domande penetranti per affinare il concetto.
3. **IL VETO:** Continuerai a discutere e a perfezionare i dettagli con l'utente finché non riceverai l'esplicito "OK, possiamo scriverlo" o il comando `/execute` o `/plan`. Solo dopo l'approvazione formale sarai autorizzato a toccare la base di codice o i documenti della Knowledge Base.
4. **FORMATTAZIONE:** Restituisci le idee in formato Markdown leggibile, usando elenchi puntati e tabelle, per facilitare la validazione visiva dell'utente.
