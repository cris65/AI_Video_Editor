# Scouting Report: Time Math Precision Bug (The 59-Minute Bleed)

## 🚨 The Twin Root Causes

The Tech Lead's hypotheses were 100% accurate. The bug is a compounding failure between the **Payload Hydration** layer and the **UI Formatting** layer. 

### Flaw 1: Payload Leak (`TimeEntryModal.tsx`)
**Location:** `frontend/src/components/tracker/TimeEntryModal.tsx` | Line 248
```tsx
    if (!isNaN(h) && !isNaN(m)) {
        d.setHours(h, m, 0, 0); // Sets to 00
        if (originalIsoString) {
            const original = new Date(originalIsoString);
            // ❌ BUG: Overwrites the 00 seconds with live timer's irregular seconds!
            d.setSeconds(original.getSeconds(), original.getMilliseconds());
        }
        return d.toISOString();
    }
```
**The Math:** If a user edits an entry originally generated by the live timer (which stores exact seconds), the `combineDateTime` function unconditionally preserves the original seconds.
E.g., `09:30:15` to `10:30:05`. 
`10:30:05 - 09:30:15` = 3590 seconds (59 minutes, 50 seconds).

### Flaw 2: Math Truncation (`useSettings.ts`)
**Location:** `frontend/src/hooks/useSettings.ts` | Line 58
```tsx
        } else {
            const h = Math.floor(seconds / 3600);
            // ❌ BUG: Floor truncates instead of rounding to nearest minute
            const m = Math.floor((seconds % 3600) / 60); 
```
**The Math:** `3590 % 3600 / 60` = 59.833 minutes. `Math.floor(59.833)` truncates strictly down to `59`, hiding the missing 10 seconds and visually displaying a 1-minute delta.

## 🛠️ The Proposed Fix
To achieve perfect mathematical parity, we must patch both sides:

1. **The Formatter (UI Layer):** 
   Update `useSettings.ts` to round duration mathematically to the nearest minute before computing the strings:
   ```tsx
   const totalMinutes = Math.round(seconds / 60);
   const h = Math.floor(totalMinutes / 60);
   const m = totalMinutes % 60;
   ```
2. **The Payload Sandbox (DB Layer):** 
   Update `combineDateTime` to only recover original seconds if the user *did not change* the hours:minutes string. If the user overrides the HH:MM input, enforce absolute zeroing (`00`).
